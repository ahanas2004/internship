# MERN Mini CRM

This is a comprehensive Mini CRM Built using the MERN stack (MongoDB, Express, React, Node.js) designed to manage Leads, Companies, and Tasks securely.

## Prerequisites

- Node.js installed
- A running MongoDB instance (or a valid MongoDB Atlas URI string)

## Setup & Running the Project Locally

### 1. Backend Setup

Open a terminal and navigate to the `backend` directory:
```bash
cd backend
```

Install the dependencies:
```bash
npm install
```

Make sure your environment variables are set in `backend/.env`. Specifically `MONGODB_URI` should be set to your correct MongoDB connection string.

Since this is a fresh setup with no users registered, you will need to populate the database with test data (Users, Companies, Leads, Tasks). Run the seed script:
```bash
npm run seed
```

Start the backend server (runs on `https://internship-anas.netlify.app`):
```bash
npm run dev
```

### 2. Frontend Setup

Open a **new** terminal window and navigate to the `frontend` directory:
```bash
cd frontend
```

Install the dependencies:
```bash
npm install
```

Start the Vite development server:
```bash
npm run dev
```

### 3. Testing the Application

Once both servers are running, access the web app by navigating to:
**http://localhost:5173**

You can use the default administrator credentials generated by the seed script.
- **Email:** `admin@minicrm.com`
- **Password:** `password123`

## Implementation Notes
- **Authorization Flow**: The application validates logins natively using `bcrypt`. Successful logins generate a stateless JWT parameter. The React frontend caches this in `localStorage` and binds it dynamically into an `Axios Interceptor` so every future HTTP call guarantees user verification against protected backend boundaries seamlessly.
- **Role Security**: Business logic blocks illegal task state updates (checking `req.user._id` natively on operations).
- **Architecture**: Separated thoroughly between modular controllers, route definitions, schemas, and a responsive functional layer on the frontend using optimized `MUI` components.
